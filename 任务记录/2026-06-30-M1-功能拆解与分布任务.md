# 当前任务：M1 功能拆解与分布任务

## 1. 用户原话

> 然后就是我会派多个 agent 干活，一个是构建 mac 端的，一个是构建 Android 端的。Android 我用的是是小米平板 6pro。先去https://www.duetdisplay.com/duet-replace-your-portable-monitor-with-software  看看这个有啥功能？我们按功能点实现我们自己的

## 2. 本轮目标

- 从 Duet 官方页面拆解功能点。
- 将功能点映射到本项目阶段。
- 明确 Mac 端 Agent 和 Android 端 Agent 的并行任务。
- 针对小米平板 6 Pro 明确 M1 显示档位。

## 3. 明确不做

- 本轮不写 Mac/Android 代码。
- 本轮不安装依赖。
- 本轮不连接 Android 设备。
- 本轮不改变 M1 冻结技术选型。

## 4. 已确认事实

- Duet 主功能包括：扩展/镜像显示、USB、Wi-Fi、触控、低延迟、低功耗、远程访问、绘图板、键鼠共享、端到端加密、隐私黑屏、ID Code 连接、重启后重连。
- 本项目当前主线是本地副屏，不做公网远程桌面和账号体系。
- M1 只验证视频链路。
- 目标 Android 设备是小米平板 6 Pro，M1 优先横屏 16:10。

## 6. 分布任务

### 解耦验证原则

Mac 端和 Android 端不能互相等待才验证。

进入端到端联调前，先完成：

1. Protocol Agent 固化 `protocol v0`。
2. Mac 端通过 H.264 文件输出或 dump client 自证。
3. Android 端通过 H.264 fixture 或 replay server 自证。
4. 双方都明确 H.264 payload 是 Annex B 还是 AVCC。

相关文件：

- `设计/M1-解耦验证策略.md`
- `测试资产/M1-解耦验证/README.md`

### Mac 端 Agent

任务目标：

- 构建 Mac Host POC。
- 从现有屏幕采集画面。
- 编码 H.264。
- 按 protocol v0 输出视频帧。
- 不等待 Android 端，先用文件输出或 dump client 验证。

必须遵守：

- 使用 Swift。
- 使用 ScreenCaptureKit。
- 使用 VideoToolbox H.264。
- 不做虚拟显示。
- 不做输入注入。
- 不引入 WebRTC。

交付物：

- Mac Host 最小可运行工程。
- 能输出 H.264 Annex B 或明确 AVCC 转换策略。
- 日志包含帧率、编码耗时、码率、错误。
- 说明需要的 macOS 权限。

验收：

- 能采集并编码本机屏幕。
- 可以向 TCP client 发送帧。
- 断开连接后停止发送或进入明确错误状态。

### Android 端 Agent

任务目标：

- 构建 Android Client POC。
- 连接 Mac Host。
- 接收 protocol v0 视频帧。
- 使用 MediaCodec 解码并渲染到 SurfaceView。
- 不等待 Mac 端，先用 fixture 或 replay server 验证。

必须遵守：

- 使用 Kotlin。
- 使用 MediaCodec。
- 使用 SurfaceView。
- M1 先横屏。
- 不做复杂 UI。
- 不做输入回传。

目标设备：

- 小米平板 6 Pro。

M1 推荐显示档位：

- 起步：1280x800 30fps。
- 稳定后：1920x1200 30fps。
- 后续优化：1920x1200 60fps。

交付物：

- Android 最小可运行工程。
- 连接状态显示。
- 解码错误显示。
- FPS 基础统计。

验收：

- 能解码测试 H.264 流。
- 能显示 Mac 端发送画面。
- Surface 生命周期切换不静默黑屏。

### Protocol Agent 或 Codex 主 Agent

任务目标：

- 固化 protocol v0。
- 保证 Mac 和 Android 字段一致。

消息类型：

- `HELLO`
- `VIDEO_CONFIG`
- `VIDEO_FRAME`
- `PING`
- `ERROR`

必须明确：

- magic。
- version。
- type。
- sequence。
- timestamp_ns。
- flags。
- payload_len。
- payload。
- 大小端。
- H.264 payload 是 Annex B 还是 AVCC。
