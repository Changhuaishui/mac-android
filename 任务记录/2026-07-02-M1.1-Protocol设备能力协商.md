## 4. 当前事实

- 已确认：
  - protocol v0 使用 TCP、32 字节 header、big endian、`HELLO` 为 Android -> Mac 的 UTF-8 JSON。
  - Android 能力识别 Agent 已新增 `DisplayCapabilities.kt`，其 JSON 结构包含 `display_capabilities`。
  - MacHost 动态原生档 Agent 的派发词要求读取 `HELLO.display_capabilities`。
- 当前假设：
  - `HELLO` JSON 增加可选字段即可满足 M1.1，不需要新增消息类型。
  - 旧 Mac Host 如只解析已知字段或忽略 HELLO payload 中未知字段，应能继续工作。
- 尚未验证：
  - Android Client 是否已经在真实 TCP `HELLO` 中发送该字段。
  - Mac Host 是否已经读取并使用该字段。
  - 真实小米平板 6 Pro 返回的运行时 mode 列表。
