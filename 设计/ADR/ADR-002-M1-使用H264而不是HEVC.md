# ADR-002：M1 使用 H.264 而不是 HEVC

日期：2026-06-30

## 状态

建议采纳

## 决策

M1 使用 H.264 作为视频编码格式。

## 原因

- Android 设备兼容性更好。
- VideoToolbox 和 MediaCodec 都成熟支持。
- 调试资料和工具更多。
- M1 关注闭环，不追求最高压缩效率。

## 替代方案

- HEVC / H.265。
- AV1。
- MJPEG。

## 暂不选择原因

- HEVC 兼容和授权环境更复杂。
- AV1 编解码延迟和设备支持不适合作为第一版。
- MJPEG 带宽太高，不适合低延迟副屏目标。

## 重新评估条件

- H.264 在目标设备上画质或带宽不可接受。
- 目标 Android 设备确认 HEVC 硬解稳定。
