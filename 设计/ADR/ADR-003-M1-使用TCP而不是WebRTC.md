# ADR-003：M1 使用 TCP 而不是 WebRTC

日期：2026-06-30

## 状态

建议采纳

## 决策

M1 使用自定义 TCP 长连接传输视频帧。

## 原因

- M1 主要在 USB 或同一局域网内验证。
- TCP 实现、调试和日志最简单。
- 可以先把问题集中在采集、编码、解码和渲染。

## 替代方案

- WebRTC。
- QUIC。
- UDP/RTP。

## 暂不选择原因

- WebRTC 工程复杂度高，会提前引入信令、ICE、拥塞控制等问题。
- QUIC/UDP/RTP 对 POC 来说调试成本偏高。

## 重新评估条件

- Wi-Fi 下 TCP 队头阻塞明显影响体验。
- 后续需要公网、NAT 穿透或弱网自适应。
