## Duet 功能点

| 功能 | Duet 页面表达 | 我们的处理 |
|---|---|---|
| 扩展显示 | Mac/PC 扩展到 iPad、Mac、PC、Android | 核心目标，M3 实现真扩展屏 |
| 镜像显示 | Second Display or Mirror | M1 先做镜像现有屏幕 |
| USB 连接 | Works over USB | M1/M2 优先支持，先用 ADB reverse 或 USB tethering |
| Wi-Fi 连接 | Works Over Wi-Fi | M4 做，M1 只保留 TCP 架构可扩展 |
| 低延迟 | zero lag、ultra-low latency、60fps | M1 指标：720p/1280x800 30fps；M4 冲 60fps |
| 低 CPU/低电量 | least CPU、least battery | M1 就使用硬件编解码，M4 做功耗优化 |
| 触控 | Touch Screen、advanced gesture control | M2 做基础触摸；高级手势后置 |
| 分辨率选项 | tons of resolution options | M4 做配置；M1 固定档位 |
| 自定义体验 | Completely Customizable | M4 做设置面板 |
| 绘图板 | graphics tablet、digital pen support、pixel perfect rendering | 后置到 M5+，不进入 M1-M3 |
| 键鼠共享 | share keyboard/mouse/trackpad between devices | 与副屏输入不同，后置；M2 只做 Android 输入回传 Mac |
| 远程桌面 | access PC/Mac remotely | 不进入当前本地副屏主线 |
| 端到端加密 | End-to-End Encryption | M5 做；M1 只保留安全底线 |
| 隐私/黑屏 | Privacy/Blackout Monitor | M5 或远程模式再评估 |
| ID & Code 连接 | Connect with ID & Code | 远程访问能力，当前不做；本地可借鉴配对码 |
| 重启后重连 | Reconnect after Reboot | 远程访问能力，当前不做；本地断线重连 M4 做 |
| 团队管理 | SSO/SAML、User management | 明确不做 |

## 我们的功能分层

### P0：必须先做

目标是证明本地副屏链路可行。

- Mac 采集现有屏幕。
- H.264 硬件编码。
- TCP 发送视频帧。
- Android 硬件解码。
- SurfaceView 显示。
- 基础 FPS、码率、错误日志。

### P1：个人可用

- Android 触摸回传 Mac。
- USB 稳定连接。
- 横屏适配小米平板 6 Pro。
- 断线可见错误。
- 手动选择分辨率和码率。

### P2：真正副屏

- macOS 虚拟显示器。
- 把窗口拖到 Android 显示。
- 坐标映射和多屏布局。
- 60fps 优化。

### P3：体验优化

- Wi-Fi 模式。
- 配对码或二维码。
- 自动重连。
- 诊断面板。
- 低功耗策略。
- 高分辨率档位。

### P4：后置能力

- 绘图板。
- 高级手势。
- 键鼠共享。
- 隐私黑屏。
- 远程桌面。
- 端到端加密。

## 明确不做

这些功能不进入当前主线：

- 账号体系。
- 公网中继。
- 团队用户管理。
- SSO/SAML。
- 商业授权。
- 远程访问他人电脑。
- 重启后远程唤醒。

## 小米平板 6 Pro 约束

按目标设备使用习惯，优先采用横屏 16:10。

M1 推荐档位：

- 起步：1280x800 30fps。
- 稳定后：1920x1200 30fps。
- 后续优化：1920x1200 60fps。
- 不在 M1 追求原生 2880x1800。

原因：

- M1 的目标是验证链路，不是榨满屏幕。
- 16:10 更贴近小米平板屏幕比例。
- 先降低码率和热量风险。

## 对架构的影响

Duet 的功能启发不改变 M1 架构。

M1 仍然是：

```text
ScreenCaptureKit
→ VideoToolbox H.264
→ TCP
→ MediaCodec
→ SurfaceView
```

进入 M2 后再扩展输入通道；进入 M3 后再处理虚拟显示。
