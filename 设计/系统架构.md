# 系统架构

## 架构目标

本项目优先做个人可用的 macOS + Android 副屏，不追求第一版商业化发行。

目标顺序：

1. 先跑通端到端视频链路。
2. 再加输入回传。
3. 最后攻克 macOS 虚拟显示。
4. 全程保持本地、安全、可验证。

## 总体结构

```text
Mac Host
  ├─ Display Source
  │   ├─ M1: 现有屏幕或窗口
  │   └─ M3: 虚拟显示器
  ├─ Capture
  │   └─ ScreenCaptureKit
  ├─ Encoder
  │   └─ VideoToolbox / VTCompressionSession
  ├─ Transport
  │   ├─ USB 调试链路 / ADB reverse
  │   ├─ USB tethering TCP
  │   └─ Wi-Fi LAN
  ├─ Control Server
  │   ├─ 配对
  │   ├─ 输入事件接收
  │   └─ 统计与诊断
  └─ Input Injector
      └─ CoreGraphics events / Accessibility

Android Client
  ├─ Discovery / Pairing
  ├─ Transport Client
  ├─ Decoder
  │   └─ MediaCodec
  ├─ Renderer
  │   └─ SurfaceView
  ├─ Input Layer
  │   ├─ Touch
  │   ├─ Mouse
  │   └─ Keyboard
  └─ Diagnostics UI

Protocol
  ├─ Video channel
  ├─ Control channel
  ├─ Input event channel
  └─ Metrics channel
```

## Mac Host 模块

### Display Source

M1 使用现有屏幕或窗口，避免被虚拟显示器卡住。

M3 再研究虚拟显示：

- 个人版：优先调查 macOS 私有虚拟显示 API 的可行性。
- 长期版：评估 DriverKit Display Driver，但不要在 M1 阶段投入。

### Capture

使用 ScreenCaptureKit 采集屏幕帧。

第一版只要求：

- 固定 1280x720 或 1920x1080。
- 30fps。
- 采集失败可重启流。

### Encoder

使用 VideoToolbox 硬件编码。

初始参数：

- Codec：H.264。
- FPS：30。
- Bitrate：8-20 Mbps。
- Low latency：关闭 B-frame 和 frame reordering。
- 队列策略：网络慢时丢旧帧，不无限排队。

### Transport

第一版使用 TCP，因为 USB 或局域网下足够简单。

后续演进：

1. TCP 分帧协议。
2. 增加关键帧请求和断线重连。
3. Wi-Fi 模式下再评估 QUIC、UDP/RTP 或 WebRTC。

### Input Injector

M2 实现：

- 点击。
- 拖拽。
- 滚轮。
- 简单键盘。

需要明确：

- macOS Accessibility 权限。
- 坐标映射。
- 多显示器坐标系。
- 输入事件日志不得记录敏感文本。

## Android Client 模块

### Decoder

使用 MediaCodec 直接解码到 Surface，减少拷贝。

第一版只支持：

- H.264 baseline/main/high 中设备可用配置。
- 固定横屏。
- 解码失败后请求关键帧。

### Renderer

使用 SurfaceView。

需要显示：

- 当前连接状态。
- FPS。
- 码率。
- 延迟估计。

### Input Layer

触摸坐标统一转换成归一化坐标：

```text
x: 0.0 - 1.0
y: 0.0 - 1.0
```

Mac Host 再映射到目标显示器坐标。

## 协议草案

### 连接握手

```text
HELLO
  protocol_version
  device_name
  client_capabilities

WELCOME
  session_id
  selected_codec
  selected_resolution
  selected_fps
```

### 视频帧

```text
VIDEO_FRAME
  stream_id
  frame_id
  timestamp_ns
  flags: keyframe / config / delta
  payload_length
  payload
```

### 输入事件

```text
INPUT_EVENT
  event_id
  timestamp_ns
  type: touch_down / touch_move / touch_up / wheel / key_down / key_up
  normalized_x
  normalized_y
  modifiers
  payload
```

## 安全边界

第一版也要遵守：

- 不接公网。
- 不上传云端。
- 不保存屏幕画面。
- 日志只保存连接、帧率、错误码、耗时。
- 连接断开后停止采集。
- 后续必须加入设备配对和加密。

## 不在第一版做

- 商业发行签名体验。
- iOS 支持。
- 多 Android 设备同时连接。
- 远程公网访问。
- 音频传输。
- 压感笔高级特性。
