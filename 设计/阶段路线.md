# 阶段路线

## M0：立项与架构

目标：

- 建立项目骨架。
- 明确 Loop 工程方式。
- 明确 Mac Host、Android Client、Protocol 三层边界。

完成标志：

- 文档入口清楚。
- 下一步 M1 可直接开任务卡。

## M1：视频链路 POC

目标：

```text
Mac 采集现有屏幕
→ H.264 编码
→ TCP 发送
→ Android 解码
→ SurfaceView 显示
```

推荐实现：

- Mac：Swift 命令行或最小菜单栏 App。
- Android：Kotlin 原生 App。
- 协议：长度前缀 + frame header + payload。
- 连接：先使用 USB 调试链路或局域网 TCP。

验收指标：

- Android 能显示 Mac 画面。
- 720p 30fps 可连续运行 10 分钟。
- 延迟、FPS、码率有基本日志。
- 解码失败能断开或恢复，不静默卡死。

## M2：输入链路

目标：

- Android 触摸回传 Mac。
- 支持点击、拖动、滚动。
- 支持基本键盘输入。

验收指标：

- 能操作 Mac 上的简单 UI。
- 坐标映射正确。
- 权限提示清楚。
- 输入日志不保存敏感文本。

## M3：虚拟显示

目标：

- Android 不只是镜像，而是成为 macOS 扩展屏。

路线：

1. 调研个人版虚拟显示 API。
2. 建立最小虚拟显示器 POC。
3. 将采集源从真实屏切换为虚拟屏。
4. 验证 macOS 显示设置、分辨率、缩放和多屏坐标。

验收指标：

- macOS 可识别一个额外显示器。
- 可以把窗口拖到虚拟显示器。
- Android 显示该虚拟屏内容。

## M4：日常可用

目标：

- 改善连接体验。
- 降低延迟与发热。
- 支持横竖屏和常用分辨率。

验收指标：

- 连续运行 1 小时。
- 断线后能重连。
- 码率可手动调节或简单自适应。
- 有诊断面板。

## M5：安全硬化

目标：

- 本地安全配对。
- 加密传输。
- 权限和日志最小化。

验收指标：

- 首次连接需要确认。
- 非配对设备不能连接。
- 日志中无画面内容、密钥、完整文本输入。
- 断连后采集和输入通道自动关闭。
