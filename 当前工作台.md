## 已知关键风险

| 风险 | 当前判断 | 处理方式 |
|---|---|---|
| macOS 真正虚拟扩展屏 | 难度最高，可能涉及私有 API 或 DriverKit | M1/M2 先不阻塞，M3 单独攻关 |
| 低延迟与发热 | 可通过硬件编解码、丢帧和码率控制优化 | M1 即加入性能指标 |
| 输入注入权限 | macOS 需要辅助功能权限 | M2 明确权限提示和安全边界 |
| Wi-Fi 抖动 | 可能导致卡顿和延迟 | 先 USB/TCP，后续再做自适应 |
| 多 Agent 方向漂移 | 容易产生范围膨胀或重复改写规则 | 用任务卡、完成定义、只读辅助 Agent 约束 |
| Mac/Android 互相等待 | 双方缺少假对端会导致无法独立验证 | protocol v0、fixture、replay server、dump client 先行 |
