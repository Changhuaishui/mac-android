# Agent 协作边界

## 总原则

一个任务只有一个主 Agent。

辅助 Agent 默认只读，除非用户明确授权其接管或修改文件。

## Codex

适合作为主 Agent：

- 工程架构设计。
- 跨 Mac、Android、协议的代码实现。
- 文件修改、测试执行、diff 检查。
- 根据证据整理任务卡和验证交付。

约束：

- 不擅自扩大范围。
- 不把 POC 写成最终安全方案。
- 不把未执行验证写成通过。
- 不覆盖用户已有修改。

## Kimi

适合辅助：

- 使用真实浏览器查看网页资料。
- 页面体验检查。
- 收集截图、页面现象和用户可见行为。
- 按清单检查交付文档是否齐全。

约束：

- 不默认承担开放式架构设计。
- 不在缺少代码证据时给最终结论。
- 不自动修改长期规则。

## Claude

适合辅助：

- 只读 Review。
- 方案第二意见。
- 检查风险遗漏。
- 长文档理解和压缩。

约束：

- 必须基于同一任务卡。
- 只报告有证据的问题。
- 不重新解释已经确认的需求。

## 人类使用者

必须由用户决定：

- 阶段目标是否改变。
- 是否允许执行高权限命令。
- 是否接受私有 API 路线。
- 是否进入 DriverKit 路线。
- 安全与体验之间的取舍。
- 是否结束任务。

## 辅助 Agent 输出格式

```text
检查对象：
发现的问题：
证据：
影响：
建议：
未检查内容：
```

辅助 Agent 的建议不是主任务结论，必须由主 Agent 或用户合并。
