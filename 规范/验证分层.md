# 验证分层

## L0：现场与变更审计

每轮默认执行：

- 当前目录结构。
- Git 状态。
- 本轮影响文件。
- 是否混入无关改动。
- 是否存在敏感信息。

## L1：静态验证

检查：

- 架构边界是否被破坏。
- 协议字段是否前后一致。
- Mac / Android 命名是否一致。
- 权限、错误和断连路径是否可见。

## L2：构建验证

检查：

- Mac Host 能编译。
- Android Client 能编译。
- 协议测试能运行。

必须在用户授权或环境允许时执行。

## L3：端到端 POC

检查：

- Mac 能采集并编码。
- Android 能接收并解码。
- 帧率、延迟、码率有数据。
- 断线或失败有可见错误。

## L4：真实设备体验

检查：

- USB 连接。
- Wi-Fi 连接。
- 横竖屏。
- 多分辨率。
- 运行 1 小时。

## L5：安全验收

检查：

- 配对。
- 加密。
- 权限最小化。
- 日志不泄露敏感内容。
- 断连自动停采集。

## 结果写法

只允许：

- 通过
- 失败
- 未验证
- 不适用

不能把“应该可以”写成“通过”。
