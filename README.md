# Mac Android 副屏项目

## 项目目标

做一个供个人使用的 macOS + Android 副屏方案，优先满足：

- 本地可控：无账号、无云服务、无遥测。
- 工程可迭代：每个阶段都有明确目标、证据、验证和复盘。
- 可分工协作：支持 Codex、Kimi、Claude 等多 Agent 共同工作，但每个任务只有一个主 Agent。
- 安全优先：屏幕采集、输入注入、网络传输和日志都遵循最小权限。

## 产品形态

```text
Mac Host
    创建或选择显示源
    采集画面
    硬件编码
    本地传输
    接收 Android 输入事件

Android Client
    连接 Mac
    硬件解码
    渲染到 Surface
    发送触摸、键盘、鼠标事件
```

## 里程碑

| 阶段 | 名称 | 目标 | 完成标志 |
|---|---|---|---|
| M0 | 立项与架构 | 明确范围、路线、Loop 工程方式 | 本目录可作为任务入口 |
| M1 | 视频链路 POC | Mac 采集现有屏幕并推送到 Android 显示 | Android 能低延迟显示 Mac 画面 |
| M2 | 输入链路 | Android 触摸/键鼠事件回传 Mac | 能点击、拖拽、滚动 |
| M3 | 虚拟显示 | Android 作为 macOS 扩展屏显示源 | macOS 能识别并输出到虚拟屏 |
| M4 | 日常可用 | USB/Wi-Fi、重连、码率、横竖屏、统计面板 | 可连续使用 1 小时 |
| M5 | 安全硬化 | 配对、加密、权限、日志策略 | 无云依赖，权限和数据边界清楚 |

## 核心原则

1. 先做闭环，再做精致。
2. 一个任务只有一个主 Agent，辅助 Agent 默认只读。
3. 不把“自动执行”当成目标；目标是让工程能持续纠错并收敛。
4. 没有证据只能写“未验证”，不能写成完成。
5. 只把重复出现、能被检查的经验升级为长期规则。
6. 技术风险以 macOS 虚拟显示器为最高优先级跟踪。
