## 实现要点

### API 兼容

| 能力 | API 30+ | API 29 fallback | 说明 |
|---|---|---|---|
| 当前 Display Mode | `display.mode` | `windowManager.defaultDisplay.mode` | API 23+ 可用 |
| supported modes | `display.supportedModes` | `windowManager.defaultDisplay.supportedModes` | API 23+ 可用 |
| window bounds | `WindowManager.currentWindowMetrics.bounds` | `Display.getSize()` | API 30 引入 WindowMetrics |
| Surface 尺寸 | `SurfaceView.width/height` | `holder.surfaceFrame` | 取有效值 |
| density | `resources.displayMetrics.density/densityDpi` | 同上 | 全版本 |
| orientation | `resources.configuration.orientation` | 同上 | 全版本 |

### 原生画质候选规则

1. 优先当前 Display Mode 的物理分辨率。
2. 如果 supported modes 中存在同分辨率多刷新率，全部列出。
3. 不强行选择 144Hz、120Hz 或 60Hz。
4. 最终发送什么分辨率/帧率由 Mac/主 Agent 决定。

### JSON 字段

字段名严格遵循 protocol-v0.md：`mode_id`、`physical_width`、`physical_height`、`refresh_rate`。不会使用 `width` / `height` 替代 `physical_width` / `physical_height`。

```json
{
  "device_name": "android tablet",
  "display_capabilities": {
    "current_mode": {
      "mode_id": 1,
      "physical_width": 0,
      "physical_height": 0,
      "refresh_rate": 0.0
    },
    "supported_modes": [
      {
        "mode_id": 1,
        "physical_width": 0,
        "physical_height": 0,
        "refresh_rate": 0.0
      }
    ],
    "window_bounds": {
      "width": 0,
      "height": 0
    },
    "surface_size": {
      "width": 0,
      "height": 0
    },
    "density": 0.0,
    "density_dpi": 0,
    "orientation": "landscape"
  }
}
```

实际运行时所有 `0` 都会被真实读取值替换。

### refresh_rate 处理规则

- `refresh_rate` 原样使用 Android `Display.Mode.getRefreshRate()` 返回的 `Float` 值。
- 转换为 JSON 时使用 `Double` 类型，不转 `Int`、不归一化到 60/120/144。
- 真实值可能不是整数，例如 `59.999004` 或 `143.998993`。
- UI 展示时通过 `DisplayCapabilities.formatRefreshRate()` 四舍五入到 1-2 位小数，例如 `59.999004` 显示为 `60`，`143.998993` 显示为 `144`。
- HELLO 消息中的 `display_capabilities` 与本 JSON 结构一致，同样保留原始 `Float/Double` 值。
- 本 Agent 只负责真实上报，不负责归一化 FPS；Mac Host 收到后按真实值决策目标 FPS。

## 需要主 Agent 合并或确认的事项

1. **协议字段建议**：VIDEO_CONFIG 后续可扩展为携带 `target_width`、`target_height`、`target_fps`，由 Mac Host 根据本 Agent 读取的 `current_mode` 决定。本 Agent 不改 protocol v0，仅输出 JSON。
2. **refresh_rate 原始值规则**：`refresh_rate` 是 Android API 返回的 float，真实值可能不是整数（例如 `59.999004` 或 `143.998993`）。本 Agent 只负责真实上报，不负责归一化 FPS；Mac Host 收到后按真实值决策目标 FPS。
3. **默认分辨率策略**：M1 目标 720p 30fps；M1.1 可在验证后升级到 2880x1800 @ 60/144Hz，需 Mac Host 编码能力配合。
4. **Surface 尺寸与物理分辨率关系**：小米平板 6 Pro 的 Surface 尺寸可能小于物理分辨率（density > 1），Mac Host 编码时应以 Surface 尺寸或当前 mode 的 physical 尺寸为参考。
