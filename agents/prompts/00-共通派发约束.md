## M1 固定范围

M1 只做：

```text
Mac 采集现有屏幕
→ H.264 编码
→ TCP 发送
→ Android 接收
→ MediaCodec 解码
→ SurfaceView 显示
```

M1 不做：

- macOS 虚拟显示。
- 输入回传。
- WebRTC。
- QUIC、UDP/RTP。
- HEVC。
- Flutter。
- Electron。
- 公网访问。
- 账号系统。
- 复杂配对和加密。
- 商业化安装体验。

## 协作边界

- 一个任务只有一个主 Agent。
- 你只能修改自己提示词允许的目录。
- 不要修改其他 Agent 的实现目录。
- 不要擅自改 protocol v0；发现协议问题时写建议，交给 Protocol Agent 或主 Agent 合并。
- 不要安装依赖、连接真实设备、修改系统权限、提交代码或发布，除非用户明确授权。
- 不要把未验证内容写成完成。
- 不要等待对端完成才验证；必须优先使用 fixture、replay server、dump client 或本地自测路径。
- 如果工具链、设备、模拟器或权限卡住，继续交付可静态检查的代码、运行说明、未验证原因和下一步，不要空等。

## 解耦验证

所有 M1 Agent 必须读取：

- `设计/M1-解耦验证策略.md`
- `测试资产/M1-解耦验证/README.md`

Mac 端必须能在没有 Android App 的情况下自证。

Android 端必须能在没有 Mac Host 的情况下自证。

端到端联调只在双方独立验证完成后进行。

## 交付格式

最终输出必须包含：

```text
完成内容：
修改文件：
执行验证：
未验证内容：
发现的问题：
需要主 Agent 合并或确认的事项：
下一步建议：
```

验证结果只允许写：

- 通过
- 失败
- 未验证
- 不适用
