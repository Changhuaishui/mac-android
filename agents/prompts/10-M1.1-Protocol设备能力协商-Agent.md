## 协议要求

1. 扩展 `HELLO` payload，增加可选字段：

```json
{
  "client_name": "Xiaomi Pad",
  "platform": "android",
  "protocol_version": 0,
  "supported_codecs": ["h264"],
  "supported_h264_stream_formats": ["annex_b"],
  "display_capabilities": {
    "current_mode": {
      "mode_id": 1,
      "physical_width": 0,
      "physical_height": 0,
      "refresh_rate": 0.0
    },
    "supported_modes": [],
    "window_bounds": {
      "width": 0,
      "height": 0
    },
    "surface_size": {
      "width": 0,
      "height": 0
    },
    "density": 0.0,
    "density_dpi": 0,
    "orientation": "landscape"
  }
}
```

2. 字段兼容规则：
   - `display_capabilities` 可缺省。
   - Mac 收不到该字段时必须降级到 `balanced` 或 `hd60`。
   - Android 不能因为 Mac 不理解该字段而断开。
3. 约束字段含义：
   - physical width/height 是 Android Display Mode 的物理模式尺寸，不是网页查到的规格。
   - refresh rate 是该 mode 对应的刷新率。
   - window/surface 尺寸只是渲染约束，不等同于设备原生分辨率。
4. 定义 Mac 选择建议：
   - `detected-native`: 使用 `current_mode.physical_width/height` 和 `current_mode.refresh_rate`，但 Mac 可因编码能力降 FPS。
   - `detected-native-safe`: 使用同分辨率，FPS capped 到 60 或更低。
   - `hd60`: 固定 1920x1200@60，作为无能力字段时的高清档。
