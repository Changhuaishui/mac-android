## 任务目标

让 Mac Host 不再硬编码“原生画质”，而是根据 Android `HELLO.display_capabilities` 生成动态档位：

- `balanced`
- `hd60`
- `detected-native-safe`
- `detected-native`

## 允许修改

- `src/mac-host/`
- `agents/reports/M1.1-MacHost动态原生档-报告.md`
- `交付记录/`
- `任务记录/`

不要修改 Android 代码。不要擅自改协议文档；如果协议字段不够，写入报告。

## 实现要求

1. Mac Host 建立 TCP 连接后读取 Android `HELLO`：
   - 如果收到 `display_capabilities`，解析并记录。
   - 如果没有收到或解析失败，降级到 `hd60` 或 `balanced`。
   - 不能因为缺少能力字段直接崩溃。
2. 实现 profile 选择：
   - `balanced`: 1280x800@30。
   - `hd60`: 1920x1200@60。
   - `detected-native-safe`: 使用 Android current mode 分辨率，FPS 上限 60。
   - `detected-native`: 使用 Android current mode 分辨率与刷新率，但允许因编码器能力降级。
3. 码率估算不能写死：
   - 基准可按 `width * height * fps * factor` 估算。
   - 设置上下限，避免离谱码率。
   - 实际公式写进报告。
4. SwiftUI App 显示：
   - Android 上报的 current mode。
   - selected profile。
   - 实际输出分辨率/FPS/码率。
   - 如果发生降级，显示降级原因。
5. CLI 支持：
   - `--profile detected-native-safe`
   - `--profile detected-native`
   - 可选 `--hello-fixture <path>` 用于无 Android 时自测 profile 选择逻辑。

## 解耦验证
