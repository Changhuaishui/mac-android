## 任务目标

为“小米 Pad 6 Pro 通过 USB 连接 Mac 做副屏”补齐电源、温控和安全策略。目标是降低长时间联调时的发热、掉电和 Mac 供电压力，并把风险可见化。

重要边界：

- App 层不能假设能控制 USB-PD 充电协商。
- 不要做任何绕过系统供电策略、root、私有 API 或危险硬件操作。
- 现代 Mac 与 Android 平板通常有 USB 供电保护；本任务目标是降低负载和清晰提示，不要宣称“绝对不会损坏硬件”。

## Android 实现要求

1. 读取并显示基础电源状态：
   - 是否正在充电
   - 电量百分比
   - 电池温度
   - 电池健康状态
   - 是否 USB 供电
2. 读取并显示热状态：
   - 优先使用 Android Thermal API。
   - 如果 API 不可用，写明限制。
3. 增加安全提示与降级建议：
   - 电池温度偏高时提示降低到 1280x800@30。
   - 电量持续下降时提示使用带供电的 USB-C hub 或给平板独立供电。
   - 解码压力过高时提示降低分辨率/帧率。
4. App 行为：
   - 连接期间保持屏幕常亮。
   - 断开后取消常亮。
   - 不在后台持续解码或保活。
5. UI 不要做复杂设置页，M1.1 只需要在主界面显示简洁状态。

## Mac/文档要求

在 README 或策略文档中写清楚推荐连接方式：

- 短时间联调：Mac 直连 Pad，使用 adb reverse。
- 长时间使用：优先使用可靠 USB-C 线材；如 Mac 掉电明显或 Pad 充电异常，改用带供电的 USB-C hub/扩展坞。
- 高画质档会增加 Mac 编码功耗与 Pad 解码功耗，不适合无限制长时间运行。
- 不建议在 Mac 电量低、机身温度高、Pad 温度高时跑设备原生档。
