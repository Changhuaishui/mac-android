## 任务目标

用 Android 原生代码读取当前平板的真实显示能力，生成“设备原生画质”的数据来源。不要使用网页规格、人工猜测或硬编码。

本 Agent 只负责“识别、展示、导出能力”，不负责 Mac 编码策略，不负责改协议字段的最终定义。

## 允许修改

- `任务记录/`

不要修改 `src/mac-host/`。不要擅自修改 `设计/protocol-v0.md`，协议字段建议写入报告。

## 实现要求

1. 新增或抽离 `DisplayCapabilities.kt`，读取：
   - 当前 display mode id
   - 当前 physical width / height
   - 当前 refresh rate
   - supported modes 列表
   - App window bounds
   - SurfaceView 实际 width / height
   - density / dpi
   - orientation
2. 对 Android API 做兼容：
   - API 30+ 优先使用 `WindowMetrics`。
   - `Display.Mode` 读取 supported modes。
   - API 差异必须有 fallback，不允许崩溃。
3. 在 App 主界面显示简洁能力摘要：
   - 当前模式：`宽x高 @ Hz`
   - 推荐原生候选：`宽x高 @ Hz`
   - App Surface：`宽x高`
4. 提供日志输出，便于用户贴回：
   - 打印一段 JSON。
   - JSON 字段命名必须稳定。
5. 推荐 JSON 结构：

```json
{
  "device_name": "android tablet",
  "display_capabilities": {
    "current_mode": {
      "mode_id": 1,
      "physical_width": 0,
      "physical_height": 0,
      "refresh_rate": 0.0
    },
    "supported_modes": [
      {
        "mode_id": 1,
        "physical_width": 0,
        "physical_height": 0,
        "refresh_rate": 0.0
      }
    ],
    "window_bounds": {
      "width": 0,
      "height": 0
    },
    "surface_size": {
      "width": 0,
      "height": 0
    },
    "density": 0.0,
    "density_dpi": 0,
    "orientation": "landscape"
  }
}
```

其中 `0` 只是示例占位，实际运行时必须是真实读取结果。

## 原生画质定义

本 Agent 输出的“原生画质候选”规则：

1. 优先使用当前 Display Mode 的 physical width / height。
2. 如果 supported modes 中存在同分辨率多刷新率，列出所有刷新率。
3. 不强行选择 144Hz、120Hz 或 60Hz。
4. 最终发送什么分辨率/帧率由 Mac/主 Agent 决定。
