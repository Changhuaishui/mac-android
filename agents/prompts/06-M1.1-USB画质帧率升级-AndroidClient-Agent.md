## 任务目标

在小米 Pad 6 Pro 真机上优化 USB 屏幕共享的显示链路，支持更高分辨率与 60fps，减少黑屏、队列堆积、延迟飘高和解码异常。

目标档位：

- 稳定档：1280x800 @ 30fps。
- 高清档：1920x1200 @ 60fps。
- 设备原生档：运行时读取小米 Pad 当前/支持的 Display Mode 后生成，不允许写死分辨率或刷新率。

不做触控、不做 Wi-Fi、不做虚拟显示、不做账号或配对。

## 实现要求

1. 根据 `VIDEO_CONFIG` 动态配置解码器：
   - width
   - height
   - fps
   - codec
   - bitrate_bps
2. 读取设备显示能力，并展示/上报：
   - 当前 Display Mode 的 physical width / height / refresh rate。
   - 所有 supported display modes。
   - App 当前 window bounds。
   - SurfaceView 实际尺寸。
   - density / dpi 信息。
   - 系统当前建议刷新率或可设置刷新率。
3. 支持解码器重建：
   - 分辨率变化时释放旧 `MediaCodec` 并重建。
   - 等待下一帧关键帧和 SPS/PPS。
   - 不允许静默黑屏。
4. 做低延迟队列策略：
   - 不无限缓存视频帧。
   - 队列积压时丢弃旧的非关键帧。
   - 关键帧和带 SPS/PPS 的配置帧优先保留。
5. UI 显示运行状态：
   - 当前分辨率
   - 目标 FPS / 实际渲染 FPS
   - 队列长度
   - 丢帧数量
   - 解码错误
   - 连接状态
   - 当前 Display Mode
   - 设备原生候选档位
6. 小米 Pad 显示适配：
   - 横屏优先。
   - SurfaceView 按视频比例居中显示，不拉伸变形。
   - 尝试设置 Surface/Window 的 frame rate 到当前视频 FPS。
   - 不把 144Hz 或任何网页规格当作默认目标；真实能力必须由代码读取。
7. 构建出的 APK 默认连接 `127.0.0.1:19421`，保持 USB reverse 路径优先。
