# Agents

本目录用于保存多 Agent 协作说明和后续可能出现的专用提示词。

当前长期规则以这些文件为准：

- `../规范/Agent协作边界.md`
- `../loop/Loop工程.md`
- `../当前工作台.md`

## 子目录

| 目录 | 用途 |
|---|---|
| `prompts/` | 可直接派发给各 Agent 的提示词 |
| `reports/` | 各 Agent 的交付报告 |
| `handoffs/` | 跨 Agent 交接材料 |

原则：

- 一个任务只有一个主 Agent。
- 辅助 Agent 默认只读。
- 辅助 Agent 输出必须带证据。
- 任务结论只能写回任务卡和验证交付，不能散落在聊天里。

派发顺序：

1. 所有 Agent 先读 `prompts/00-共通派发约束.md`。
2. 再读自己的角色提示词。
3. 完成后写入 `reports/`。
4. 主 Agent 统一合并到任务卡、交付记录和当前工作台。
