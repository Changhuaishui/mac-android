# 验证交付：M1.1 Android 设备显示能力识别

## 1. 本次改动

- 解决的问题：Android Client 之前没有读取真实设备显示能力的入口，Mac Host 无法基于设备原生参数决策编码分辨率/帧率。
- 实际改动：
  - 新增 `DisplayCapabilities.kt`，读取 `current_mode`、`supported_modes`、刷新率、`window_bounds`、Surface 尺寸、`density`、`orientation`。
  - 在 `MainActivity` 中展示能力摘要，并在 logcat 输出稳定 JSON。
  - 新增 UI TextView 与字符串资源。
- 影响范围：仅 `src/android-client/`，不影响 Mac Host、Protocol、现有 TCP/解码逻辑。
- 明确未改动：
  - `src/mac-host/`
  - `src/protocol/`
  - `设计/protocol-v0.md`
  - `TcpClient.kt`、`VideoDecoder.kt`、`AssetPlayer.kt`、`Protocol.kt`

## 2. 运行入口

- Android Client：打开 App 即自动读取显示能力，无需额外操作。
- 日志 Tag：`MacDisplayCapabilities`
- 日志内容：JSON 字符串（格式化后 2 空格缩进）。

## 3. 触发条件

- 前置环境：Android 设备/模拟器运行 App。
- 必要权限：无需额外权限。
- 连接方式：不需要连接 Mac Host；能力识别为纯本地读取。
- 操作步骤：启动 App，查看右上角能力摘要；或运行 `adb logcat -s MacDisplayCapabilities:D`。

## 4. 验证步骤

1. 在 Android Studio 打开 `src/android-client`。
2. 点击 Build → Make Project（或运行 `./gradlew :app:assembleDebug`）。
3. 连接小米平板 6 Pro（或启动模拟器）。
4. 点击 Run，启动 App。
5. 观察右上角能力摘要是否显示。
6. 在 Terminal 执行：
   ```bash
   adb logcat -s MacDisplayCapabilities:D
   ```
7. 复制输出的 JSON，粘贴到本报告和 `agents/reports/M1.1-Android设备显示能力识别-报告.md`。

## 5. 预期结果

- Android：App 启动后右上角显示类似：
  ```text
  当前模式：2880x1800 @ 144Hz
  推荐原生候选：2880x1800 @ 144Hz
  App Surface：1800x2880
  ```
  注意：UI 中的刷新率已经过 1-2 位小数格式化；logcat JSON 中保留 Android `Display.Mode.getRefreshRate()` 原始 Float/Double 值（可能不是整数，如 `143.998993`）。
- 网络：HELLO 消息会携带 `display_capabilities` JSON，`refresh_rate` 同样保留原始 Float/Double 值。
- 安全：不上传、不保存任何屏幕内容；仅读取本地显示参数。
