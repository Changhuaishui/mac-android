# M1.1 Mac Host 动态原生档 验证说明

日期：2026-07-01
任务：M1.1 Mac Host 动态原生档（根据 Android 能力选择输出档位）
Agent：Codex (M1.1 MacHost 动态原生档 Agent)

## 验证目标

1. Mac Host 能解析 Android HELLO 中的 `display_capabilities`，优先使用 `physical_width` / `physical_height`，兼容旧 fixture 的 `width` / `height`。
2. `refresh_rate` 按容差归一化后再参与档位选择，日志同时输出 `raw_refresh_rate`、`normalized_fps`、`selected_fps`。
3. 支持 `balanced`、`hd60`、`detected-native-safe`、`detected-native` 四个 profile。
3. 无 HELLO 或解析失败时降级到 `hd60`，并记录原因。
4. 码率按 `width * height * fps * factor` 估算，且有上下限。
5. CLI 支持 `--profile` 与 `--hello-fixture`。
6. SwiftUI App 显示 current mode、selected profile、实际输出、降级原因。
7. TCP 模式收到 HELLO 后按 profile 推流；无 HELLO 时 5 秒超时降级。

## 验证环境

- macOS（当前开发机）
- Swift 5.9+ / Xcode Command Line Tools
- 已授予 `machost` 屏幕录制权限
- 无 Android Client 连接
- 使用 Python 3 编写假 Android Client 验证 TCP 接收

## 复现命令

```bash
cd src/mac-host
swift build

# balanced
./.build/debug/machost --profile balanced --dump /tmp/balanced.h264 --dump-duration 3

# hd60
./.build/debug/machost --profile hd60 --dump /tmp/hd60.h264 --dump-duration 3

# detected-native-safe（使用 fixture 自测）
./.build/debug/machost \
  --profile detected-native-safe \
  --hello-fixture ../../测试资产/M1-解耦验证/hello-display-capabilities.example.json \
  --dump /tmp/native-safe.h264 --dump-duration 3

# detected-native（使用 fixture 自测）
./.build/debug/machost \
  --profile detected-native \
  --hello-fixture ../../测试资产/M1-解耦验证/hello-display-capabilities.example.json \
  --dump /tmp/native.h264 --dump-duration 3

# 无 fixture 的降级路径
./.build/debug/machost --profile detected-native-safe --dump /tmp/native-safe-degrade.h264 --dump-duration 2
```

## TCP 验证脚本

```bash
./.build/debug/machost --profile detected-native-safe --port 19422
```

使用 Python 假 Android Client 连接 `::1:19422`，发送 HELLO（含 `display_capabilities`），即可收到 `VIDEO_CONFIG` 与 `VIDEO_FRAME`。

## 未验证原因

1. 当前运行环境为无图形界面的 shell，无法对 SwiftUI 窗口进行鼠标点击和视觉确认。
2. Android Client 尚未实现 `display_capabilities` 真实上报。
3. TCP server 仍为 Network.framework IPv6-only，未替换为双栈 POSIX socket server。
