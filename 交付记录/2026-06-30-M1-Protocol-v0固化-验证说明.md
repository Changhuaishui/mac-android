# 验证交付：M1 Protocol v0 固化

## 1. 本次改动

- 解决的问题：Mac Host 与 Android Client 缺少一份可共同实现的 protocol v0 固化文档。
- 实际改动：新增协议规范、协议目录入口、Protocol Agent 报告和本验证交付记录。
- 影响范围：仅协议文档、协议目录、任务/报告/交付记录和当前工作台。
- 明确未改动：未修改 Mac Host、Android Client、AGENTS.md、M1 冻结技术选型。

## 2. 运行入口

- Mac Host：不适用。
- Android Client：不适用。
- Protocol：`设计/protocol-v0.md`、`src/protocol/README.md`。
- 文档入口：`agents/reports/M1-Protocol-Agent-报告.md`。

## 3. 触发条件

- 前置环境：阅读 `设计/protocol-v0.md`。
- 必要权限：不适用。
- 连接方式：M1 后续使用 TCP 长连接。
- 操作步骤：Mac/Android Agent 按协议文档实现 header 编解码和消息处理。

## 4. 验证步骤

1. 检查协议文档是否包含 magic、version、type、sequence、timestamp_ns、flags、payload_len、payload。
2. 检查大小端、H.264 payload 格式、SPS/PPS 策略、keyframe 标识和错误码是否明确。
3. 检查 Swift/Kotlin 字段映射建议是否与固定 header 一致。
4. 检查本轮是否修改了禁止修改目录。

## 5. 预期结果

- Mac：能按 `设计/protocol-v0.md` 实现 header 写入和 Annex B 视频发送。
- Android：能按 `设计/protocol-v0.md` 实现 header 读取和 Annex B 视频解码输入。
- 网络：TCP 接收端可按 32 字节 header + payload_len 处理粘包。
- 安全：协议不引入云服务、账号、遥测或屏幕保存。
