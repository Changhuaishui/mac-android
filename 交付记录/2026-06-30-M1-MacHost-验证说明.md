Agent：Codex (M1 MacHost Agent)

## 验证目标

- Mac Host 可编译运行。
- 协议行为对齐 `设计/protocol-v0.md`：
  - Mac 不主动发送 HELLO。
  - `VIDEO_CONFIG` 为 UTF-8 JSON，不含二进制 SPS/PPS。
  - 关键帧 `VIDEO_FRAME` 携带 SPS/PPS 并设置正确 flags。
- 新增 `--dump` 本地自测路径：capture→encode 后输出 Annex B `.h264` 文件。
- TCP server 可被 IPv6 dump client 验证。

## 验证环境

- macOS（当前开发机）
- Swift 5.9+ / Xcode Command Line Tools
- 已授予 `machost` 屏幕录制权限
- 无 Android Client 连接

## 关键修复

- 编码器无输出问题的根因：`VTCompressionSessionCreate` 未声明 `imageBufferAttributes`。
- 修复方式：在 `imageBufferAttributes` 中指定 `kCVPixelBufferPixelFormatTypeKey = 420YpCbCr8BiPlanarFullRange`，并补全 width/height。
- 修复后 `Encoder` 输出回调正常触发，可产出 Annex B H.264 帧。

## 复现命令

### dump 自测

```bash
cd src/mac-host
swift build
./.build/debug/machost --dump /tmp/sample-annexb.h264 --dump-duration 5
xxd -l 16 /tmp/sample-annexb.h264
```

### TCP + IPv6 dump client

```bash
# 终端 1
./.build/debug/machost

# 终端 2
python3 -c "
import socket, time
s = socket.socket(socket.AF_INET6, socket.SOCK_STREAM)
s.connect(('::1', 19421))
s.settimeout(6)
open('/tmp/capture_tcp.h264','wb').write(s.recv(1024*1024))
print('done')
"
```
